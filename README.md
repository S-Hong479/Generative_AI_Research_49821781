# Using Generative AI Tools - Boon or Bane

## Project Overview

This repository stores research materials for the project **Using Generative AI Tools - Boon or Bane**. It is organised to support efficient research data management, version control, backup, and collaboration among research team members.

## Repository Structure

### `Literature_Review/`
Contains literature review materials such as journal articles, conference papers, books, white papers, comparative studies, and background readings.

### `Quantitative_Analysis/`
Contains survey questions, raw or cleaned survey datasets, CSV files, Excel files, Python scripts, charts, and quantitative analysis reports.

### `Qualitative_Analysis/`
Contains interview transcripts, consent forms, interview summaries, qualitative analysis files, and qualitative visualisations. Some files in this folder may contain sensitive participant information and should only be accessed by authorised team members.

### `Drafts_and_Reports/`
Contains draft research proposals, draft reports, conference paper drafts, final reports, and other written research outputs.

### `Additional_Materials/`
Contains information sheets, photos, posters, workshop materials, media files, and other supporting project materials.

## Naming Convention

A clear naming convention helps team members identify the content, date, version, and purpose of each file. Recommended format:

`YYYYMMDD_description_version_author`

Example:

`20260427_survey_data_cleaned_v01_Sage.csv`

## Collaboration Guidelines

1. Pull the latest version of the repository before making changes.
2. Create a new branch for major edits or new analysis work.
3. Use meaningful commit messages that clearly describe the change.
4. Do not overwrite another team member's work.
5. Use pull requests for review before merging major changes into the main branch.
6. Keep confidential and sensitive data private.
7. Only authorised team members should access consent forms, interview transcripts, raw participant responses, or files containing identifiable participant information.

## Example Commit Messages

- Created initial research folder structure
- Added literature review materials

## Data Access Control

Sensitive research files should be stored in a private repository or in restricted-access storage. These include consent forms, raw survey responses, interview transcripts, and any documents containing participant information. Access should be limited to authorised project members only.
