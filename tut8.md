
Activity 1

Q1.
Efficient file storage and organisation are essential, as they enable research team members to quickly locate, understand and reuse project materials. A clear structure also reduces the risk of losing files, copying outdated versions or using incorrect data during analysis. In collaborative research projects, good organisation enhances transparency and makes it easier for future researchers to review the work or continue the research.

Q2.
In this project, literature review documents should be stored in a clearly labelled ‘Literature’ folder, whilst quantitative data, Python scripts, CSV files, Excel files and analysis reports should be stored in a separate ‘Analysis’ folder. Qualitative materials such as interview transcripts, informed consent forms and interview protocols should be stored separately, as they may contain personal or sensitive information. Files containing participant information, informed consent forms, raw survey responses and interview transcripts should be subject to special access controls, such as private repository access and restricted permissions.

Activity 2
Q1.
A good naming convention is important because it makes files easier to identify, search, and manage throughout a research project. Consistent file names reduce confusion between different versions and help team members understand the file content without opening it. This is especially useful in collaborative research because it supports version control, reproducibility, and long-term data management.
